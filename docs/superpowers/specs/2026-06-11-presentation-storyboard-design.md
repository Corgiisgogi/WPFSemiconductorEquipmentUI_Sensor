# 발표 PPTX 스토리보드 설계 — 반도체 장비 센서 제어 UI

- 작성일: 2026-06-11
- 산출물: 발표용 PPTX (PowerPoint, 16:9)
- 상태: 스토리보드 승인 완료 → 레이아웃 계획 → 제작

## 1. 목표와 제약

| 항목 | 결정 |
|------|------|
| 청중/자리 | 교수·심사위원 (교육과정 최종 평가) |
| 톤 | 문제 인식 → 근거 있는 설계 → 결과. 기술적 엄밀성 + 내러티브 |
| 발표 시간 | 약 15분 (일부 슬라이드는 발표 중 생략 가능하되, **PPT에는 모든 내용 포함**) |
| 언어 | 한국어 (기술 용어는 영문 병기 허용) |
| 분량 | 약 22장 (내용 완비형) |
| 내러티브 구조 | **B안: 문제→해결 내러티브** |
| 폰트 | **Pretendard 패밀리만 사용** (Thin/ExtraLight/Light/Regular/Medium/SemiBold/Bold/ExtraBold/Black). Pretendard에 이탤릭이 없으므로 이탤릭 미사용 |
| 스타일 | 모던 (여백 충분, 그리드 정렬, 절제된 색, 굵기 대비로 위계 표현) |

## 2. 포함 섹션 결정

- 포함: 표준 소개(개요·아키텍처·핵심기능·기술스택) + **UI 화면 데모(스크린샷)** + **트러블슈팅·회고** + **팀 역할분담·협업**
- 제외(독립 섹션으로는): 자체 코드 점검·품질 개선 → 단, 트러블슈팅·회고 슬라이드에 일부 녹임
- 보류 항목(보고서/발표에서 "트레이너 데모용 설계"로 명시): AI 자동제어가 안전 인터록을 우회하는 점은 하드웨어 제어 한계상 의도된 동작

## 4. 슬라이드별 스토리보드 (22장)

1. **표지** — 프로젝트명, 부제(WPF 반도체 장비 센서 트레이너), 팀명·발표자, 날짜 / 대표 화면 썸네일

### Part 2. 문제 정의 & 배경 (3)
4. **배경** — 반도체 장비 센서 모니터링 교육 환경, 실장비 없이 학습 필요성 / 개념 일러스트
5. **문제 정의 & 목표** — 실시간 센서 감시·위험 자동대응·이력관리·접근통제 / 목표 카드

### Part 3. 설계 의사결정 (3)
7. **기술 스택 & 선택 근거** — .NET 4.7.2·WPF·MVVM(무프레임워크)·SQLite·TwinCAT ADS·Flask + 이유 / 스택 다이어그램
8. **시스템 아키텍처** — Views→ViewModels→Services→Models, 합성 루트(MainViewModel) / 아키텍처 다이어그램

### Part 4. 구현 — 핵심 기능 (7)
